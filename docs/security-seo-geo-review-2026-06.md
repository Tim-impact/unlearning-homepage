# 언러닝컴퍼니 홈페이지 — 보안 점검 & SEO/GEO 개선 리포트

> **목적**: 게시판(Story) 도입 이후 진행한 보안 점검과 SEO/GEO 개선 작업을 팀에 공유하기 위한 기록 문서
> **대상**: 언러닝컴퍼니 팀 (기술·비기술 모두)
> **작성일**: 2026-06-29
> **대상 사이트**: https://un-learning.co (정적 `index.html` SPA + Vercel 배포 + Sanity 헤드리스 CMS)

---

## 0. 한눈에 보기 (TL;DR)

- **보안: 심각한 위험 없음.** 게시판의 핵심 위험인 "외부인의 무단 글 작성/수정"은 차단되어 있음을 실제로 확인했습니다. 추가로 방어를 강화하는 권고 3건도 조치 완료.
- **SEO/GEO: 게시글이 검색·AI 검색에 잘 노출되도록 4가지를 개선.** 가장 중요한 건, 그동안 **게시글 본문이 검색 로봇에게 안 보이던 문제**를 해결한 것입니다.
- 모든 변경은 **운영 사이트에 배포 후 실제로 검증**(서버 응답·브라우저 네트워크 확인)했습니다.

---

## 1. 배경

홈페이지에 **Story 게시판**(News·Blog)을 도입하면서, 글은 **Sanity**라는 헤드리스 CMS(콘텐츠 관리 도구)로 관리하고 홈페이지가 이를 불러와 보여주는 구조가 되었습니다.

게시판이 생기면 외부에서 입력·조회되는 영역이 늘어나므로, **보안 점검**과 함께 게시글이 검색에 잘 잡히도록 하는 **SEO/GEO 점검**이 필요했습니다.

- **SEO**(Search Engine Optimization): 구글 등 검색엔진에 잘 노출되게 하는 최적화
- **GEO**(Generative Engine Optimization): ChatGPT·Claude·Perplexity 같은 **생성형 AI 검색**이 우리 콘텐츠를 잘 읽고 인용하게 하는 최적화

## 2. 보안 점검

### 2.1 점검 방법
코드(클라이언트·서버리스 함수·배포 설정)를 직접 검토하고, **운영 서버에 실제 요청을 보내** 동작을 확인했습니다(증거 기반).

### 2.2 결과 — 이상 없음 ✅

| 점검 항목 | 결과 | 설명 |
|---|---|---|
| **게시판 무단 쓰기 차단** (가장 중요) | ✅ 안전 | 인증(토큰) 없이 글 생성을 시도하니 **403 거부**("permission required"). 데이터셋은 **읽기만 공개, 쓰기는 토큰 필수**. 외부인이 글을 만들거나 고칠 수 없음 |
| **비밀키·토큰 노출** | ✅ 없음 | 홈페이지 코드·함수·설정에 비밀키 없음. 노출된 값(Sanity 프로젝트 ID, Web3Forms·Google Analytics 키)은 모두 **공개되어도 안전한 공개키** |
| **XSS(악성 스크립트 삽입)** | ✅ 차단 | 게시글 본문은 React가 자동으로 안전 처리(이스케이프). 서버 함수도 출력값을 이스케이프. 원시 HTML을 그대로 삽입하는 경로 없음 |
| **보안 헤더** | ✅ 구비 | CSP, HSTS, X-Frame-Options(DENY), X-Content-Type-Options, Referrer-Policy, Permissions-Policy 적용 |

> **핵심 메시지**: 게시판이 추가됐지만, 외부인이 글을 조작하거나 사이트를 변조할 수 있는 경로는 없습니다.

### 2.3 조치한 보안 강화 (권고 → 적용 완료)

발견된 "낮은 위험" 권고사항을 선제적으로 조치했습니다.

| 강화 항목 | 무엇을 | 효과 |
|---|---|---|
| **본문 링크 스킴 검증** | 게시글 본문 링크 중 `http(s)`·`mailto`만 실제 링크로 표시하고, `javascript:` 같은 위험한 링크는 일반 텍스트로 처리 | 만에 하나 악성 링크가 입력돼도 클릭 시 스크립트가 실행되지 않음 |
| **CSP 연결 도메인 화이트리스트** | 사이트가 데이터를 주고받을 수 있는 외부 주소를 "모든 https" → **Sanity·Google Analytics·Web3Forms로 한정** | 공격이 일어나도 데이터가 임의의 외부로 빠져나갈 통로를 좁힘 |
| **함수 입력값 정리** | 게시글 주소(번호/ID) 입력에서 따옴표·역슬래시 제거 | 데이터 조회 쿼리에 대한 인젝션 방지 |

### 2.4 남은 권고 (의도적 보류)
- **CSP `'unsafe-eval'`**: 현재 사이트는 브라우저에서 화면 코드를 즉석 변환하는 방식(Babel)을 써서 이 옵션이 필요합니다. 보안 강도를 더 높이려면 향후 **빌드 단계(사전 컴파일)** 도입이 필요하며, 이는 별도 구조 변경 작업입니다. (지금 당장의 위험은 낮음)

## 3. SEO / GEO 개선

### 3.1 진단 — 무엇이 문제였나
게시글 페이지는 **자바스크립트가 실행된 뒤에야 본문이 그려지는** 구조였습니다. 구글은 자바스크립트를 실행하지만, **상당수의 AI 검색 로봇·소셜 미리보기 봇은 자바스크립트를 실행하지 않아 본문을 통째로 못 읽는** 상태였습니다. 또한 게시글마다 설명문이 똑같았고, 사이트맵·구조화 데이터도 미비했습니다.

### 3.2 조치 (4건, 모두 적용·검증 완료)

| 개선 | 무엇을 했나 | 왜 / 효과 |
|---|---|---|
| **① 본문을 HTML에 포함** | 게시글 주소 접속 시, 서버가 **본문 텍스트와 구조화 데이터를 HTML에 함께 넣어** 내려주도록 함 (`<noscript>` + JSON-LD) | 자바스크립트를 안 돌리는 **AI 검색·크롤러도 본문을 읽음** → GEO 핵심 개선. 사람이 보는 화면은 그대로 |
| **② 구조화 데이터(JSON-LD)** | 각 글에 `BlogPosting` 스키마(제목·발행일·이미지·작성자·본문) 삽입 | 구글 리치결과·AI 인용에 유리. 기계가 "이건 언제 발행된, 누구의, 무슨 글"인지 명확히 이해 |
| **③ 글별 메타 설명** | 공유·검색 시 보이는 설명문을 **통일 문구 → 글 본문 요약**으로 변경 | 글마다 고유한 설명 → 검색엔진의 "중복 설명" 약점 해소, 클릭률 향상 |
| **④ 동적 사이트맵** | `sitemap.xml`이 **모든 게시글 주소를 자동 포함**하도록 함(글 추가 시 자동 반영) | 검색엔진이 새 글을 빠르게 발견 |

> 참고: `robots.txt`는 이미 일반 크롤러뿐 아니라 **생성형 AI 봇(GPTBot·ClaudeBot·PerplexityBot·Google-Extended)을 명시 허용**하도록 잘 설정돼 있었습니다.

### 3.3 검증
- 게시글 주소(`/post/1`) 응답에 **본문 텍스트·BlogPosting 구조화데이터·글별 설명**이 실제로 포함됨을 확인
- `sitemap.xml`에 정적 페이지 + 게시글이 포함됨을 확인
- 사람이 보는 화면은 변화·중복·오류 없음(콘솔 에러 0)

## 4. 함께 진행된 게시판 기반 작업 (맥락)

이번 점검 전후로 게시판 사용성·노출 관련 작업도 함께 진행되었습니다(참고).

- **깔끔한 주소**: 게시글 URL을 난수 → **작성순 고정 번호**(`/post/1`)로, 주소창의 `#`도 제거(`/post/1`, `/news` 등). 한번 부여된 번호는 글 삭제에도 안 바뀌어 **공유 링크가 안정적**.
- **공유 미리보기(OG)**: 카카오톡·페이스북 등에서 글을 공유하면 **글별 제목·대표이미지·설명**이 뜨도록 서버 함수로 구현.
- **읽기 경험**: 본문 가독성(행간·제목 크기·이미지 전체 노출), 목록 카드 정리, 본문 끝 "공유 넛징" 등.
- **Google Analytics(GA4)** 설치 — 방문·페이지뷰 집계(화면 전환까지 자동 추적).

## 5. 팀이 알아두면 좋은 운영 포인트

- **새 글을 올리면**: 주소·번호·사이트맵 포함은 **자동**입니다. 별도 설정 불필요.
- **글을 빨리 검색에 띄우고 싶으면**: Google Search Console → URL 검사에 해당 글 주소(`https://un-learning.co/post/N`) 입력 → "색인 생성 요청".
- **공유는 글의 "공유 버튼"으로**: 주소창의 `…/post/N`을 그대로 복사해도 되지만, 공유 버튼이 미리보기가 보장되는 주소를 복사해 줍니다.
- **태그 관리**: 게시판 태그(공지·프로그램·후기·칼럼·인터뷰·사례·일기)는 Sanity Studio 설계에 정의되어 있으며, 변경하려면 코드 수정 + Studio 재배포가 필요합니다(요청 시 지원).
- **방문 성과 보기**: Google Analytics → 실시간/보고서. (표준 보고서는 집계까지 약 24시간)
- **공유 미리보기가 옛 내용으로 보이면**: 카카오톡 등은 캐시를 합니다. 카카오 OG 캐시 초기화 도구로 해당 주소를 갱신하면 됩니다.

## 6. 용어 설명 (비개발자용)

- **SEO**: 구글 등 검색에 잘 나오게 하는 최적화
- **GEO**: ChatGPT·Claude 같은 AI 검색이 콘텐츠를 잘 읽고 인용하게 하는 최적화
- **OG(Open Graph)**: 링크 공유 시 뜨는 미리보기(제목·이미지·설명) 정보
- **CSP(Content-Security-Policy)**: 브라우저에 "이 사이트는 이런 외부 자원만 허용"이라고 알려 공격을 줄이는 보안 규칙
- **JSON-LD / 구조화 데이터**: 페이지 내용을 기계가 이해하도록 정리한 데이터(예: 이건 글, 발행일은 언제…)
- **사이트맵**: 사이트의 모든 페이지 목록. 검색엔진이 페이지를 발견하는 지도
- **헤드리스 CMS(Sanity)**: 화면과 분리된, 글을 쓰고 저장하는 콘텐츠 관리 도구

- **SEO/GEO**: 게시글이 일반 검색과 AI 검색 양쪽에 제대로 노출될 기반을 마련했습니다(특히 "본문이 안 보이던" 문제 해결).
