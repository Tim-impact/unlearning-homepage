# "Why We Started" 풀블리드 선언문 밴드 — 설계

**날짜:** 2026-06-20
**대상 파일:** `index.html` (단일 파일 React SPA)
**범위:** About 섹션(`#about`) 맨 아래 `# TEAM` 카드 블록을, "왜 이 일을 시작했는가" 선언문 밴드로 교체

---

## 1. 목표

About 섹션 끝의 팀 카드(3명)는 "팀 정보가 확 튀는" 느낌이라 제거한다. 그 자리에
**우리가 왜 이 일을 시작했는지**를 하나의 서사 선언문으로 담아, 섹션을 감정적으로
닫고 바로 다음 Contact 섹션("함께 도전하고 싶은 일이 있다면")으로 자연스럽게
잇는다.

위쪽 구성(헤딩 · Vision/Mission · `# OUR_PHILOSOPHY` · `# HOW_WE_WORK`)은
**변경하지 않는다.**

## 2. 핵심 메시지 (왜)

- **근본 원인 = '연결의 부재'**: 일 맡길 조직엔 자원이 없고, 일 배울 청년에겐
  현장이 없다.
- **우리의 해법**: 그 둘을 *하나의 프로젝트 안에서* 만나게 한다 → 일은 더 잘
  굴러가고, 사람은 현장에서 자란다.
- **오퍼레이션에 진심인 이유**: 그것이 결국 *사람을 남기는 일*이기 때문.
- **추진체 메타포**: 사회혁신 조직(로켓)이 미션(궤도)에 오르도록 돕는
  추진체(언러닝컴퍼니).

> 집중 분야(임팩트 커리어 · 소규모 조직 AX 전환 · 커뮤니티)와 핵심
> 콘텐츠(AX 기반 운영 방법론)는 *무엇을*에 해당하므로 이 밴드에 넣지 않는다.
> (위쪽 섹션의 역할)

## 3. 확정 카피 (서사형)

라벨: `# WHY_WE_STARTED`

> 문제의 뿌리는 하나였습니다 — **연결의 부재.**
>
> 일 맡길 조직엔 자원이 없고,
> 일 배울 청년에겐 현장이 없었습니다.
>
> 우리는 그 둘을 한 프로젝트 안에서 만나게 합니다.
> 일은 더 잘 굴러가고, 사람은 현장에서 자랍니다.
>
> 오퍼레이션에 진심인 이유는 분명합니다.
> **그것이 결국, 사람을 남기는 일이기 때문입니다.**
>
> — 조직이라는 로켓이 미션이라는 궤도에 오르도록,
> 우리는 곁에서 일하는 추진체가 되기로 했습니다.

문구는 구현 중 미세 조정 가능. 구조(진단 → 해법 → 사람 → 추진체)는 고정.

## 4. 시각 설계 (통일감)

### 4.1 풀블리드 다크 밴드 (기존 패턴 재사용)
- Work 섹션의 `.work-hero-band` 기법을 그대로 사용:
  `position: relative; left: 50%; margin-left: -50vw; width: 100vw; overflow: hidden`.
  → About은 전체가 밝은 soft 톤이므로, 마지막에 다크 밴드가 나오면 명도 대비로
  시선이 모이고 섹션이 단단히 닫힌다.
- 배경: 기존 `.aurora` 3겹(green `#00ff88` · cyan `#22d3ee` · blue `#2563eb`
  drift)을 재사용 → Work 섹션 다크 밴드와 **시각적 운율(visual rhyme)**.
- 로켓 메타포가 Work 섹션의 실제 로켓·궤도·지구 애니메이션과 같으므로, 언어로만
  메타포를 반복해도 위쪽 애니메이션과 의미가 연결된다. (밴드 자체에는 별도
  로켓 그래픽을 넣지 않는다 — 중복·과잉 방지.)

### 4.2 타이포그래피 (철학 카드와 차별화)
철학 블록 = "작은 인용구 + 긴 본문 카드". 이 밴드 = "큰 선언문".
- 라벨: JetBrains Mono(MONO), green, `letter-spacing .1em`.
- 선언문 본문: Paperlogy(HEAD), `clamp(22px, 3.4vw, 36px)`, 흰색(textBright),
  가운데 정렬, `line-height 1.55`, `word-break: keep-all`.
- 강조 줄("연결의 부재", "사람을 남기는 일", 추진체 줄)은 굵게/포인트색으로
  대비.
- 마지막 추진체 줄은 약간 작게(보조 톤)으로 마무리.

### 4.3 레이아웃
- 밴드 내부 콘텐츠는 중앙 정렬, `max-width` ~840px, 좌우/상하 넉넉한 패딩
  (데스크톱 상하 ~96px, 모바일 ~64px).

## 5. 모션 & 접근성

- 오로라 드리프트는 기존 `@keyframes aurora-1/2/3` 재사용.
- `@media (prefers-reduced-motion: reduce)` 규칙에 이미 `.aurora`가 포함돼 있어
  모션 최소화가 자동 적용된다. (신규 추가 시 같은 규칙에 포함.)
- 스크롤 진입 페이드업: 기존 `useInView` + `STYLES.fadeIn` 재사용.

## 6. 반응형

- 풀블리드 유지, 내부 패딩만 축소. 가로 스크롤 0은 기존 밴드와 동일 기법이라
  보장.
- 선언문은 `clamp()`로 자동 스케일, `word-break: keep-all`로 한글 줄바꿈 보호.

- `AboutSection` 내 `<TeamCards visible={visible} />` 호출 (~1634행).
- `CONTENT.about.team` 데이터 배열 (~384–406행).
- 더 이상 참조되지 않으면 `.team-photo` CSS 규칙(122행 내) 정리 검토.
- 팀 사진 에셋(`assets/team-*.png`)은 별도 확인 후 제거 (이 설계 범위 밖,
  사용자 확인).

**추가**
- About 섹션 끝에 `WhyWeStartedBand` 컴포넌트 (풀블리드 다크 밴드 + aurora +
  선언문).
- 필요 시 CSS: 기존 `.work-hero-band` / `.aurora`를 공용 클래스로 재사용하거나
  About용 래퍼 클래스 추가(동일 속성).
- `CONTENT.about`에 선언문 카피 필드 추가(예: `whyLabel`, `whyBody`).

**유지**
- 헤딩 · subtext · Vision/Mission · PhilosophyBlock · HOW_WE_WORK 전부 그대로.

## 8. 완료 기준 (Definition of Done)

- [ ] About 섹션에 팀 카드 3명이 더 이상 보이지 않는다.
- [ ] 그 자리에 풀블리드 다크 선언문 밴드가 렌더된다.
- [ ] 밴드 배경 오로라가 기존 Work 밴드와 동일 톤으로 움직인다.
- [ ] 데스크톱/모바일 모두 가로 스크롤이 발생하지 않는다.
- [ ] `prefers-reduced-motion` 환경에서 오로라가 멈춘다.
- [ ] 선언문 카피가 확정안과 일치하고 한글 줄바꿈이 깨지지 않는다.
- [ ] 제거된 `team` 데이터/컴포넌트 참조가 코드에 남아있지 않다.
